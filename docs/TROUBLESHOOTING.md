# Troubleshooting

This guide collects first-run failures observed while preparing ConvexKit, plus the most common setup issues for this stack.

## `npm run typecheck` Fails With `Cannot find type definition file for 'vite/client'`

Cause: dependencies are not installed.

Fix:

```bash
npm install
npm run typecheck
```

In CI or Codespaces, prefer:

```bash
npm ci
```

## `npm test` Fails With `vitest: command not found`

Cause: dependencies are not installed.

Fix:

```bash
npm install
npm test
```

## `.env.local` Is Missing

Run the setup wizard:

```bash
npm run setup
```

The wizard creates `.env.local`, generates `BETTER_AUTH_SECRET`, and can optionally set Convex backend env vars.

## Convex URLs Are Missing Or Invalid

Symptoms:

- The app fails during startup with invalid environment variables.
- The browser console mentions `VITE_CONVEX_URL`.

Fix:

1. Run `npx convex dev` to create or connect a Convex deployment.
2. Copy the `.convex.cloud` URL into `VITE_CONVEX_URL`.
3. Copy the `.convex.site` URL into `VITE_CONVEX_SITE_URL`.
4. Re-run `npm run dev`.

## Google Sign-In Does Not Appear

Google OAuth is optional and hidden by default.

To enable it:

```bash
VITE_GOOGLE_AUTH_ENABLED=true
GOOGLE_CLIENT_ID=your-client-id.apps.googleusercontent.com
GOOGLE_CLIENT_SECRET=your-client-secret
```

Then set the backend env vars:

```bash
npx convex env set GOOGLE_CLIENT_ID "$GOOGLE_CLIENT_ID"
npx convex env set GOOGLE_CLIENT_SECRET "$GOOGLE_CLIENT_SECRET"
```

Anonymous realtime messages work without Google OAuth.

## Protected Routes Redirect To Home

Protected routes such as file uploads require an authenticated session. If Google OAuth is disabled, those routes are intentionally unavailable from the default anonymous demo.

## Convex Backend Env Vars Are Missing

Symptoms:

- Convex logs warn about missing env vars.
- Auth callbacks fail after enabling OAuth.

Fix:

```bash
npx convex env set BETTER_AUTH_SECRET "$BETTER_AUTH_SECRET"
npx convex env set SITE_URL "$SITE_URL"
```

If Google OAuth is enabled:

```bash
npx convex env set GOOGLE_CLIENT_ID "$GOOGLE_CLIENT_ID"
npx convex env set GOOGLE_CLIENT_SECRET "$GOOGLE_CLIENT_SECRET"
```

## Wrangler Deploy Cannot Find `dist/server/wrangler.json`

Cause: deploy config is generated by the build.

Fix:

```bash
npm run build:preview
npm run sync:wrangler-config
```

Then deploy:

```bash
npx wrangler deploy --config dist/server/wrangler.json
```

## Cloudflare Deploy Fails With Authentication Errors

Confirm these are set in your shell, `.env.local`, or GitHub Actions secrets:

```bash
CLOUDFLARE_API_TOKEN=...
CLOUDFLARE_ACCOUNT_ID=...
```

For GitHub Actions preview deploys, also set:

```bash
VITE_CONVEX_URL=...
VITE_CONVEX_SITE_URL=...
CONVEX_DEPLOY_KEY_PREVIEW=...
```

## Generated Convex Types Are Missing

Symptoms:

- Imports from `@convex/_generated/api` fail.
- TypeScript cannot find generated Convex modules.

Fix:

```bash
npx convex dev --once
```

Generated files live under `convex/_generated/` and are intentionally ignored by git.
